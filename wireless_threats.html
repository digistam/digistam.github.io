Wireless Threats

Wireless networks can be at risk to various types of attacks, including:

      access-control attacks
      integrity attacks
      confidentiality attacks
      availability attacks
      authentication attacks

Access Control Attacks

Wireless access-control attacks aim to penetrate a network by evading WLAN access-control measures, such as AP MAC filters and Wi-Fi port access controls.

      WarDriving: WLANs detected by sending probe requests over a connection or by listening to web beacons; Tools: KisMAC, NetStumbler
      Rogue access points: in order to create a backdoor to a trusted network, an attacker may install an unsecured AP or fake AP inside a firewall; A wireless AP is termed a rogue access point when it is installed on a trusted network without authorization
      MAC spoofing: reconfigure a MAC address to appear as an authorized AP to a host on a trusted network; Tools: SMAC
      AP misconfiguration
      Ad hoc associations: using any USB adapter or wireless card; connecting host to an unsecured client to attack a specific client or to avoid AP security
      Promiscuous client: using an promiscuous client, an attacker exploits the behavior of 802.11 wireless cards as they always attempt to find a stronger signal signal to connect; Intent to lure the client to connect to the attacker's AP
      Client mis-association: client may intentionally of accidentally connect or associate with an AP outside the legitimate network because the WLAN signals travel through the air, walls and other obstructs
      Unauthorized association

Integrity Attacks

An integrity attack involves changing or altering data during transmission. In wireless integrity attacks, attackers send forged control, management or data frames over a wireless network to misdirect wireless devices and perform another type of attack such as a DoS attack.

      Data-Frame injection: constructing and sending forged 802.11 frames; Tools: Airpwn, File2air, Wperf, void11, WEPWedgie, wnet dinject
      WEP injections: constructing and sending forged WEP encryption keys; Tools: WEP cracking and injection tools
      Bit-Flipping attacks: capturing frame and flipping random bits in the data payload, modifying ICV and sending it to user
      Extensible AP replay: capturing 802.1X Extensible Authentication Protocols (eg. EAP Identity, Success and Failure) for later replay; Tools: wireless capture and injection tools between client and AP
      Data Replay: capturing 802.11 data frames for later (modified) replay; Tools: capture and injection tools
      Initialization Vector Replay Attacks: deriving the keystream by sending a plaintext message
      RADIUS Replay: capturing RADIUS Access-Accept or Reject messages for later replay; Tools: Ethernet capture and injection tools between AP and authentication server
      Wireless Network Viruses: viruses can provide an attacker with a simple method to compromise APs

Confidentiality Attacks

These attacks attempt to intercept confidential information sent over a wireless network, regardless of whether the system transmits data in cleartext or an encrypted format. If the system transmits data in an encrypted format (like WEP or WPA), an attacker may attempt to break the encryption.

      Eavesdropping: capturing and decoding unprotected application traffic to obtain potentially sensitive information; Tools: Wireshark, Ettercap, Bettercap, Kismet, commercial analyzers
      Traffic Analysis: inferring information from the observation of external traffic characteristics
      Cracking WEP Key: capturing data to recover a WEP key using brute force or Fluhrer-Mantin-Shamir (FMS) cryptanalysis; Tools: Aircrack-ng, AirSnort, chopchop, WepAttack, WebDecrypt
      Evil Twin AP: posing as an authorized AP by beaconing the WLAN's SSID to lure users; Tools: HostAP, EvilTwinFramework, Wifiphisher
      Honeypot AP: setting an AP's SSID to be the same as that of a legitimate AP; Tools: manipulating SSID
      Session Hijacking: manipulating the network such that the attacker's host appears to be the desired destination; Tools: manipulating
      Masquerading: pretending to be an authorized user to gain access to a system; Tools: stealing login IDs and passwords, bypassing authentication mechanisms
      MITM attack: running conventional MITM attack tools on an evil twin AP to intercept TCP sessions or Secure Sockets Layer (SSL) / Secure Shell (SSH) tunnels; Tools: dsniff, Ettercap, aLTEr attack

Availability Attacks

Availability attacks aim at obstructing the delivery of wireless services to legitimate users, either by crippling WLAN resources or by denying them access to these resources.

      Access Point Theft: physicall removing an AP
      Disassociation Attacks: destroying connectivity, making unavailable
      EAP-Failure: observing a valid 802.1X EAP exchange and then sending the client a forged EAP-Failure message; Tools: File2air, Airtool Pi
      Beacon Flood: generating thousands of counterfeit 802.11 beacons to make it difficult for clients to find a legitimate AP; Tools: FakeAP
      Denial-of-Service: exploiting the carrier-sense multiple access with collision avoidance (CSMA/CA) clear channel assessment (CCA) mechanism to make a channel appear busy; Tools: an adapter that supports the CW Tx mode, with a low level utility to invoke continuous tranmissions
      De-authenticate Flood: flooding client(s) with forged de-authenticates or disassociates to disconnect users from an AP; Tools: AirJack, void11
      Routing Attacks: distributing routing information within the network; Tools: RIP protocol, exploiting Ad-Hoc On-Demand Distance Vector (AODV) and Dynamic Source Routing (DSR) protocols using wormhole and sinkhole attacks
      Authenticate Flood: sending forged authenticates or associates from random MACs to fill a target APs association table; Tools: Airjack, File2Air, void11
      Address Resolution Protocol (ARP) Cache Poisoning Attacks: creating many attack vectors
      Power Saving Attacks: transmitting a spoofed traffic indication map (TIM) or delivery TIM (DTIM) to a client in the power-saving mode, making the client vulnerable to a DoS attack
      TKIP MIC Exploit: generating invalid TKIP data to exceed the target AP's MIC error threshold, suspending WLAN service; Tools: File2air, wnet dinject

Authentication Attacks

Object is to steal the identity of Wi-Fi clients, their personal information, login credentials etc. to gain unauthorized access to network resources.

      PSK Cracking: recovering a WPA PSK from captured key handshake frames using a dictionary attack tool; Tools: Cowpatty, KisMAC, Fern Wifi Cracker
      LEAP Cracking: recovering user credentials from captured 802.1X Lightweight EAP (LEAP) packets using a dictionary attack tool to crack the NT password hash; Tools: Asleap, THCLEAPcracker
      VPN Login Cracking: gaining user credentials (eg. Point-to-Point Tunneling Protocol (PPTP) password or Internet Protocol Security (IPsec) pre-shared secret key) using brute force attacks on virtual private network authentication protocols; Tools: ike-scan, IKECrack (IPsec), Anger, THC-pptp-bruter (PPTP)
      Domain Login Cracking: recovering user credentials (eg. Windows login and password) by cracking NetBIOS password hashes with a brute-force or dictionary-attack tool; Tools: John the Ripper, L0phtCrack, THC-Hydra
      Key Reinstallation Attack: exploiting the four-way handshake of the WPA2 protocol; Tools: None reuse technique
      Identity Theft: capturing user identities from cleartext 802.1X Identity Response packets; Tools: packet capturing tools
      Shared Key Guessing: attempting 802.11 shared key authentication with the vendor default or cracked WEP keys; Tools: WEP cracking tools, Wifite
      Password speculation: repeatedly attempting 802.1X authentication using a captured identity to guess the user's password; Tools: password dictionary
      Application Login Theft: capturing user credentials (eg. email address and password) from cleartext application protocols; Tools: Ace Password Sniffer, dsniff, Wi-Jacking Attack
