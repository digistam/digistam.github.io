NTFS streams

Hide malicious files

Hiding files in NTFS streams only works with NTFS filesystems.

What's the use ? Attackers may hide malicious files in the target system and keep them invisible from the legitimate users by using NTFS streams, and may remotely execute them whenever required.

How does it work?

      Open dosbox (cmd)
      Create c:\magic
      Create a text file (readme.txt) using Notepad
      Copy calc.exe from c:\Windows\System32 to c:\magic
      Command: "type c:\magic\calc.exe > c:\magic\readme.txt:calc.exe"
      Now, calc.exe will be hidden in readme.txt (the size of readme.txt will not be altered)
      Delete calc.exe from the c:\magic folder
      Command: "mklink backdoor.exe readme.txt:calc.exe"
      You need Administrator rights to do this
      Command: "backdoor.exe" will run the calculator program
