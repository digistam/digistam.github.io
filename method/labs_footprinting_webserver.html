Footprinting web server

Footprinting is an essential task for any pentester. Before proceeding to hack or exploit a webserver, the pentester must know the type and version of the webserver as most of the attacks and exploits are specific to type and version of the server being used by the target. By performing web server footprinting it is possible to gather valuable system-level data such as account details, OS, software versions, server names and database schema details.

Ghost Eye

Ghost Eye is an information gathering tool written in Python3. Input is a domain or IP. Ghost Eye gathers information such as Whois lookup, DNS lookup, EtherApe, Nmap port scan, HTTP header grabber, Clickjacking test, Robots.txt scanner, Link grabber, IP location finder and traceroute.

      pip3 install -r requirements.txt
      python3 ghost_eye.py
      Perform Whois Lookup: 3
      Enter a domain and view the results
      Perform DNS Lookup: 2
      Perform Clickjacking test: 6
      By performing this test, GhostEye will provide the complete architecture of the web server and also reveal whether the domain is vulnerable to ClickJacking attacks or not
      You can also use Nmap port scan, HTTP header grabber, link grabber etc.

Perform web server reconnaissance using Skipfish

Skipfish is an active web application (deployed on a webserver) security reconnaissance tool. It prepares an interactive sitemap for the targeted site by carrying out a recursive crawl and dictionary-based probes. The resulting map is then annotated with the output from a number of active security checks.

      Start Wampserver64 on a 'victim' system
      From the 'attacker' system:
      skipfish -o /home/attacker/test -S /usr/share/skipfish/dictionaries/complete.wl http://[IP Address of victim system]:8080
      On receiving this command, Skipfish performs a heavy brute-force attack on the web server by using the complete.wl dictionary file, creates a directory named test in the root location and stores the result in index.html inside this location
      On completion of the scan, Skipfish generates a report and stores it in the test directory
      Open the report index.html in a web browser and examine the vulnerabilities found

Footprint a web server using the httprecon tool

httprecon is a tool for advanced web server fingerprinting. This tool performs banner-grabbing attacks, status code enumeration and header ordering analysis on its target web server.

      Run httprecon.exe as administrator
      Enter website url and port (80) and click Analyze
      A footprint of the website appears
      The Get existing tab shows the server (nginx)
      When attackers obtain this information, they research the vulnerabilities present in nginx and try to exploit them
      Click the GET long request tab, which lists all GET requests
      Click the Fingerprint Details tab
      The details show name of the protocol the website is using and its version
      By obtaining this information, attackers can manipulate HTTP vulnerabilities in order to perform malicious activities such as sniffing over the HTTP channel which might result in revealing sensitive data such as user credentials

Footprint a web server using Netcat and Telnet

      nc -vv www.moviescope.com 80
      netcat will display the hosting information of the provided domain
      GET / HTTP/1.0
      Netcat will perform the banner grabbing and gather information such as content type, last modified data, accept ranges, ETag and server information

      The same can be done with Telnet
      telnet www.moviescope.com 80
      GET / HTTP/1.0
      Telnet will perform banner grabbing and gather information such as content type, last modified date, accept ranges, ETag and server information
