Wireshark remote sniffing

Wireshark is a network packet analyzer used to capture network packets and display packet data in detail. The tool uses Winpcap to capture packets on its own supported networks. It captures live network traffic from Ethernet, IEEE 802.11, PPP/HDLC, ATM, Bluetooth, USB, Token Ring, Frame Relay, and FDDI networks. The captured files can be programmatically edited via the command-line. A set of filters for customized data displays can be refined using a display filter. Here, we will use the Wireshark tool to perform password sniffing.

HTTP POST

In the "apply a display filter" field, type http.request.method == POST to filter only http POST traffic packets

      Navigate to Edit -> Find Packet from the menu bar
      Click Display filter, select String from the dropdown options, click Narrow & Wide and select Narrow (UTF-8 / ASCII)
      Click Packet list, select Packet details from the dropdown options
      In the field next to String, type pwd and click the Find button
      Expand the HTML Form URL Encoded: application/x-www-form-urlencoded node

Remote Interfaces

      Click Capture in the menu bar -> Options
      In the popup screen, click Manage Interfaces
      The Manage Interfaces window appears
      Click the Remote Interfaces tab
      Click + to add a remote host and its interface
      The Remote Interface window appears
      In the Host text field, enter the IP address of the target machine
      In the Port field, enter the port number as 2002
      Under the Authentication section, select the Password authentication radio button and enter the target machine's user credentials
      A new remote interface is added to the Manage Interfaces window, click OK
      The newly added remote interface appears in the Wireshark Capture Options window, click Start

In real-time, when attackers gain the credentials of a victim's machine, they attempt to capture its remote interface and monitor the traffic its user browses to reveal confidential user information

Prepare victims computer

      Start remote desktop connection to victim's computer
      Start Control Panel on the remote computer
      Navigate to System and Security -> Windows Tools
      In the Windows Tools control panel, double click Services
      In the Services window, choose Remote Packet Capture Protocol v.0 (experimental)
      Click Start
