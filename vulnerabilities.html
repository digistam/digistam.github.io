    Vulnerability Management Life Cycle

    Important process that helps identify and remediate security weaknesses before they can be exploited.

    Pre-Assessment Phase

        prepatory phase, defining policies and standards, clarifying scope of the assessment and creating a good baseline for vulnerability management.
        Identify and understand business processes
        Identify the applications, data and services that support the business processes and perform code reviews
        Identify the approved software, drivers and basic config for each system
        Create an inventory of all assets and prioritize or rank the critical assets
        Understand the network architecture and map the network infrastructure
        Identify the controls already in place
        understand policy implementation and practice standard compliance with business processes
        Define the scope of the assessment
        Create information protection procedures to support effective planning, scheduling, coordination and logistics
        Classify the identified assets according to the business needs. Classification helps to identify the high business risks in an organization. Prioritize the rated assets based on the impact of their failure and their reliability in the business.
        Prioritization helps:
        -> Evaluate and decide a solution for the consequence of the assets failing
        -> Examine the risk tolerance level
        -> Organize methods for prioritizing the assets

    Vulnerability Assessment Phase

        Crucial phase in vulnerability management
        Examine and evaluate the physical security
        Check for misconfigurations and human errors
        Run vulnerability scans using tools
        Select the type of scan based on the organization or compliance requirements
        Identify and prioritize vulnerabilities
        Identify false positives and false negatives
        Apply the business and technology context to scanner results
        Perform OSINT information gathering to validate the vulnerabilities
        Create a vulnerability scan report

    Post Assessment Phase

        Recommendation phase
        Creating a priority list for assessment recommendations based on the impact analysis
        Developing an action plan to implement the proposed remediation
        Capturing lessons learned to improve the complete process in the future
        Conducting training for employees
        Risk Assessment -> risks are identified, characterized and classified along with the techniques used to control or reduce their impact. Risk levels can be high, moderate or low.
        Remediation -> applying fixes, based on risk ranking, develop and action plan to implement the remediation, perform root-cause analysis, apply patches and fixes, capture lessons learned, conduct awareness training, perform exception handling and risk acceptance for the vulnerabilities that can not be remediated
        Verification -> re-scan of systems to assess if the remediation is complete, verification of remedies, using ticketing systems, scanners and reports
        Monitoring -> periodic vulnerability scan and assessment, timely remediation of identified vulnerabilities, monitoring intrusion detection, implementing policies, procedures and controls

    Vulnerability Classification and Assessment Types

    Vulnerability Classification

    Misconfigurations / Weak configurations:

        Most common vulnerability, mainly caused by human error
        Attackers detect misconfigurations through various scanning techniques
        Default configurations of devices have to be changed and optimized

    Network Misconfigurations

        Frequent changes to network and security devices are inevitable and essential for business improvement
        Insecure protocols -> transmitting information or data in plaintext without encryption, use of vulnerable protocols (ftp, telnet etc.)
        Open Ports and Services -> Servers often operate with some ports open, but all open ports are not dangerous, unless they are misconfigured, unpatched or implemented with poor security rules. Open ports must be limited. Leaving ports open for unnecessary services can invite new threads to the network, they may lead to the loss of data or DDOS attacks.
        Errors -> improper configuration of applications or services can generate error reports, outdated software can also generate security errors, invite code injections. Applications should not disclose critical information that can help attackers.
        Weak encryption -> can allow attackers to perform man-in-the-middle attacks, sniff the traffic, masquerade as legitimate service to communicate with end users with false information, same goes for key generation with guessable credentials, insecure key distribution

    Host Misconfigurations

        Open Permissions -> granting unnecessary permissions to a user or group can lead to security issues such as data leakage or corruption of system functionality, an attacker can perform privilege escalation by using unnecessarily created accounts to access unprotected files or to run command on the operating system
        Unsecured Root Accounts -> using manufacturer-allotted default administrative account credentials can lead to system security issues, attackers can guess the credentials using different brute-force techniques

    Application Flaws

    Application flaws are vulnerabilities in applications that are exploited by attackers. Applications should be secured using the validation and authorization of the user.

        Buffer Overflows
        Memory Leaks
        Resource Exhaustion
        Integer Overflows
        Null Pointer / Object Dereference
        DLL Injection
        Race Conditions
        Time of Check / Time of Use

    Improper Input Handling

    Verification of application functionalities such as validation, filtering, sanitizing, encryption and decryption of INPUT data.
