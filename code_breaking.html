Code breaking methodologies

Brute Force

Brute force attacks, or exhaustive attacks, determine the keys by trying every possible combination of characters. The efficiency depends on the hardware configured. The use of faster processors means that more keys will be tested per second. Cryptanalysts carried out a succesful brute force attack on a DES encryption method, which effectively rendered DES obsolete.

It is extremely difficult to crack cryptographic systems, as they have no practival weaknesses to exploit; however, it is not impossible. Cryptographic systems use cryptographic algorithms to encrypt a message. These cryptographic algorithms use a key to encrypt or decrypt messages. In cryptography, this key is the important parameter that specifies the transformation of plaintext to ciphertext and vice versa. If you are able to guess or find the key used for decryption, then you can decrypt the messages and read them in clear text. 128-bit keys are common and considered strong. From a security perspective, to avoid guessing the key, cryptographic systems use randomly generated keys. However, you still have a choice to determine the key used for encryption or decryption.

You can attempt to decrypt a message using all possible keys until you discover the key used for encryption. This method of discovering is called brute force attack. Doing so requires a massive amount of processing power. It is resource intensive and time intensive. The average time needed to find the key in a brute-force attack depends on the length of the key. How longer the key, how longer it takes to brute force attack it. Difficulty depends on:

      Length of the key
      Number of possible values each component of the key can have
      Time it takes to attempt each key
      If there is any mechanism that locks the attacker out after a certain number of failed attempts
