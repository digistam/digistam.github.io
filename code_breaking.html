Code breaking methodologies

Brute Force

Brute force attacks, or exhaustive attacks, determine the keys by trying every possible combination of characters. The efficiency depends on the hardware configured. The use of faster processors means that more keys will be tested per second. Cryptanalysts carried out a succesful brute force attack on a DES encryption method, which effectively rendered DES obsolete.

It is extremely difficult to crack cryptographic systems, as they have no practival weaknesses to exploit; however, it is not impossible. Cryptographic systems use cryptographic algorithms to encrypt a message. These cryptographic algorithms use a key to encrypt or decrypt messages. In cryptography, this key is the important parameter that specifies the transformation of plaintext to ciphertext and vice versa. If you are able to guess or find the key used for decryption, then you can decrypt the messages and read them in clear text. 128-bit keys are common and considered strong. From a security perspective, to avoid guessing the key, cryptographic systems use randomly generated keys. However, you still have a choice to determine the key used for encryption or decryption.

You can attempt to decrypt a message using all possible keys until you discover the key used for encryption. This method of discovering is called brute force attack. Doing so requires a massive amount of processing power. It is resource intensive and time intensive. The average time needed to find the key in a brute-force attack depends on the length of the key. How longer the key, how longer it takes to brute force attack it. Difficulty depends on:

      Length of the key
      Number of possible values each component of the key can have
      Time it takes to attempt each key
      If there is any mechanism that locks the attacker out after a certain number of failed attempts

Frequency Analysis

      Study of frequency of letters or groups of letters in a ciphertext
      In any given stretch of written language, certain letters and combinations of letters occur with varying frequencies
      This technique examines the number of times that a particular symbol appears in a ciphertext
      Example: the letter e is a common letter in the English language; If the letter k appears commonly in a ciphertext, it can be reasonably concluded that k in the encrypted language is equivalent to e in English
      Encrypted source code is more vulnerable to these types of attacks because words such as #define, struct, else and return are repeated frequently in code

Trickery and Deceit

      Requires high level of mathematical and cryptographic skills
      Social engineering
      It is fairly easy to decrypt an entire message if the user knows some of its content

One-Time Pad

One can crack any cipher if provided with sufficient time and resources. An exception is One-Time Pad, which is assumed unbreakable even with infinite resources. A one-time pad mostly contains a non-repeating set of letters or numbers, chosed by the system randomly. The user writes them on small sheets of paper and then pastes them together in a pad. The sender encrypts only one plaintext character using each key letter on the pad, and the receiver decrypts each letter of the ciphertext using an identical pad. Once the letter uses a page, he tears it off the pad and securely discards it; hence the name one-time pad. A drawback is that the key length is the same as that of the message, thus making it impossible to encrypt and send large messages.

Cryptography Attacks

A cryptography attack or cryptanalysis involves the study of various principles and methods of decrypting the ciphertext back to the plaintext without knowledge of the key.

Ciphertext-only attack

      Less effective but much more likely for the attacker
      The attacker only has access to a collection of ciphertexts
      The attack is completely succesful if the corresponding plaintexts (or even better, the key) can be deduced
      Obtaining any information at all about the underlying plaintext is considered a success
      Often, the result of an attack is just a partial break and not a complete break

Adaptive Chosen-plaintext attack

      The attacker has complete access to the plaintext message including its encryption
      He can modify the content of the message by making a series of interactive queries, choosing subsequent plaintext blocks based on the information from the previous encryption queries and functions
      To perform this attack, an attacker needs to interact with the encryption device

Chosen-plaintext attack

      Highly effective type of cryptanalysis attack
      The attacker obtains the ciphertexts corresponding to a set of plaintexts of his/her own choosing
      The attacker attempts to derive the key used and thus decrypt other messages encrypted with that key
      Since the attacker knows the plaintext and the resultant ciphertext, he gains many insights into the key used
      This technique can be difficult but is not impossible

Related-Key attack

      Similar to chosen plaintext attack, except that the attacker can obtain ciphertexts encrypted under two different keys
      Very useful attack is you can obtain the plaintext and matching ciphertext
      The attack requires that the differing keys be closely related, e.g. in a wireless environment where subsequent keys might be derived from previous keys
      The keys are different, but close
      Most likely resulting in a partial break

Dictionary attack

      After building a dictionary, if the attacker obtains the ciphertext, the attacker uses the already built dictionary to find the corresponding plaintext
      Decrypting keys, passwords, passphrases and ciphertext

Known-plaintext attack

      The only information available to the attacker is some plaintext blocks along with the corresponding ciphertext and algorithm used to encrypt and decrypt the text
      Using this information, the key used to generate the ciphertext is deduced so as to decipher other messages
      This attack works on block ciphers and is an example of linear cryptanalysis
      The known plaintext blocks are generated using a series of intelligent guesses and logic, and not by accessing the plaintext over a channel

Chosen-ciphertext attack

      The attacker obtains the plaintext corresponding to an arbitrary set of ciphertexts of his own choosing
      Using this information, the attacker tries to recover the key used to encrypt the plaintext
      To perform this attack, the attacker must have access to the communication channel between the sender and the receiver
      Two variants:
      Lunchtime of Midnight attack -> the attacker can have access to the system for only a limited amount of time or can access only a few plaintext-ciphertext pairs
      Adaptive Chosen-ciphertext attack -> the attacker selects a series of ciphertexts and then observes the resulting plaintext blocks

Rubber Hose attack

using coercion or torture -> people under pressure can't maintain security

Chosen-key attack

      An attacker not only breaks a ciphertext but also breaks into a larger system, dependent of that ciphertext
      Once an attacker breaks the cipher, he gets access to the system, and he can control the whole system

Timing attack

      Based on repeatedly measuring the exact execution times of modular exponentiation operations
      The attacker tries to break the ciphertext by analyzing the time take to execute the encryption and decryption algorithm for various inputs
      In a computer, the time taken to execute a logical operation may vary based on the input given
      An attacker tries to extract the plaintext by giving varying inputs

Man-in-the-Middle attack

      Attack performed against a cryptographic protocol
      An attacker intercepts the communcation between a client and a server and negotiates the cryptographic parameters
      Using this attack, an attacker can decrypt the encrypted content and obtain confidential information such as system passwords
      An attacker can also inject commands that can modify the data in transit
      MITM is usually performed on public key cryptosystems where key exchange is required before communication takes place

Birthday Attac

A birthday attack refers to a class of brute-force attacks against cryptographic hashes that renders brute-forcing easier to perform. This attack depends on the birthday paradox which is the probability of two or more people in a group of 23 sharing the same birthday is greater than 0.5.

How many people are needed to have a high likelihood that two will share the same birthday. There are 365 days a year, and therefore you might think that at least half or 182 people share the same birthday, when it is actually only 23! Even with 23 people in a room, there is a 50% chance that two of them will have their birthdays on the same day and month.

The idea behind the birthday attack is to attempt to find a collision for a given hash. Assume the hash is MD5 with 128-bit output. You would have to try 2^128 possible hashes to guarentee a collision, which is a VERY LARGE number. From the birthday paradox, we still need a very large number but many orders of magnitude smaller than the first value.

Meet-in-the-Middle attack on digital signature schemes

A meet-in-the-middle attack is the best attack method for cryptographic algorithms using multiple keys for encryption. This attack reduces the number of brute-force permutations required to decode text encrypted by more than one key. A meet-in-the-middle attack uses space-time trade-off; it is also a type of birthday attack because it exploits the mathematics behind the birthday paradox, and the attack consumes less time than an exhaustive attack. It is called a meet-in-the-middle attack because it works by encrypting from one end and decrypting from the other end, thereby meeting in the middle.

In the meet-in-the-middle attack, the attacker uses a known plaintext message. The attacker has access to both the plaintext as well as the respective encrypted text. This attack is performed by attackers for forging messages that use multiple encryption schemes.

Side-Channel attack

A side-channel attack is a physical attack performed on a cryptographic device/cryptosystem to gain sensitive information. Cryptography is generally part of the hardware or software that runs on physical devices such as semi-conductors (resistor, transistor etc.) that interact with and affect various environmental factors as follows:

      Power Consumption -> simple power analysis (SPA), differential power analysis (DPA)
      Electromagnetical field -> computer components often generate electromagnetic radiation; by measuring the variations of the electromagnetical field over the chip surface, an attacker can predict its correlation to the underlying computation and data and may be able to deduce some valuable information about this computation and data
      Light emission -> deduce the data a computer is processing based on the optical radiation emitted from its LED status indicators
      Timing and Delay
      Sound -> acoustic attacks exploit the sound produced during a computation; these acoustic emissions are from keyboards and computing components (CPU, memory etc.)

Hash collision attack

      Performed by finding two different input messages that result in the same hash output
      hash(a1) = hash(a2)
      Since the algorithm itself randomly selects these messages, attackers have no role in the content of these messages. This allows the attacker to perform cryptanalysis by exploiting the digital signature used to generate a different message with the same hash value
      Once the attacker detects a collision in the hash, he can identify more collisions by concatenating the data to matching messages

DUHK attack

Don't Use Hard-Coded Keys (DUHK) is a cryptographic vulnerability that allows attackers to obtain encryption keys used to secure VPNs and web sessions. This attack mainly affects any hardware / software using the ANSI X9.31 Random Number Generator (RNG). Pseudorandom number generators (PRNGs) generate random sequences of bits based on the initial secret value, called seed and the current state. The PRNG algorithm generates cryptographic keys that are used to establish a secure communication channel over the VPN. In some cases, the seed key is hardcoded into the implementation. Both the factors are key issues of the DUHK attack, as any attacker can combine ANSO X9.31 with the hard coded seed key to decrypt the encrypted data sent or received by that device.

MITM attackers use the DUHK attack to learn the seed value, observe the current session, and obtain the current state value. Using this attack, attackers can identify encryption keys and steal confidential information such as critical business data, user credentials and credit card details.
