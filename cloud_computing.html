Cloud Computing

Cloud computing is an emerging technology that delivers computing services, such as online business applications, online data storage and webmail over the internet. The cloud offers benefits but also poses many threats and risks to organizations.

Cloud computing is an on-demand delivery of IT capabilities, in which IT infrastructure and applications are provided to subscribers as metered services over networks.

      On-demand self-service
      Distributed storage
      Rapid elasticity
      Automated management
      Broad network access
      Resource pooling
      Measured service
      Virtualization technology

Limitations

      Limited control and flexibility of organizations
      Proneness to outages and other technical issues
      Security, privacy and compliance issues
      Contracts and lock-ins
      Dependence on network connections
      Potential vulnerability to attacks as every component is online
      Difficulty in migrating from one service provider to another

Infrastructure-as-a-Service IaaS

On-demand fundamental IT resources (computing power, virtualization, data storage, network, virtual machines and other abstracted hardware and OSs

Advantages: dynamic infrastructure scaling, guaranteed uptime, elastic load balancing, global accessability

Disadvantages: software security at high risk (third party providers), performance issues and slow connection speeds

Platform-as-a-Service PaaS

Development of applications and services, subscribers need not buy and manage the softwware and infrastructure underneath it, Google App Engine, Microsoft Azure

Advantages: simplified deployment, prebuilt business functionality, lower security risks compared to IaaS, instant community, scalable

Disadvantes: vendor lock in, data privacy, integration

Software-as-a-Service SaaS

Application software on-demand over the internet, Google Docs, Salesforce CRM

Advantages: low cost, easy administration, global accessability, high compatibility

Disadvantes: security and latency issues, total dependency on the internet, switching between vendors is difficult

Identity-as-a-Service IDaaS

Authentication services to subscribed enterprises, it provides services such as Single-Sign-On (SSO), Multi-Factor-Authentication (MFA), OneLogin, Microsoft Azure Active Directory

Advantages: low cost, improved security, simplify compliance, reduced time, central management of user accounts

Disadvantages: single server failure may disrupt the service or create redundancy on other authentication servers, vulnerable to account hijacking attacks

Security-as-a-Services SECaaS

Integrates security services into corporate infrastructure, developed based on SaaS and does not require any physical hardware

Advantages: low cost, reduced complexity, continuous protection, improved security through best security practice, latest and updated security tools, rapid user provisioning, agility

Disadvantages: increased attack surfaces and vulnerabilities, unknown risk profile, insecure APIs, vulnerable to account hijacking attacks

Container-as-a-Service Caas

Provides containers and clusters as a service to its subscribers, virtualization of container engines, management of containers, applications and clusters through a web portal or API, Amazon EC2, Google Kubernetes Engine GKE

Advantages: streamlined development of containerized applications, pay-per-resource, low cost, few resources, improved security, high scalability

Disadvantages: high operational overhead, platform development is the developer's responsibility

Function-as-a-Service FaaS

Platform for developing, running and managing application functionalities, mostly used while developing applications for microservices, Amazon AWS, Google Cloud Functions, Microsoft Azure Functions

Advantages: pay-per-use, low cost, efficient security updates, easy deployment, high scalability

Disadvantages: high latency, memory limitations, monitoring and debugging limitations, unstable tools and framework, vendor lock-in

Anything-as-a-Service XaaS

Anything, including digital products such as tools, applications, technologies, food, transportation, medical consults etc., NetApp, AWS Elastic Beanstalk, Heroku

Advantages: highly scalable, independent of location and devices, fault tolerance and reduced redundancy, reduced capital expenditure, enhances business process

Disadvantages: chances of service outage as XaaS is dependent on the internet, performance issues, highly complex

Firewall-as-a-Service FWaaS

Filtering network traffic, enhanced data analysis capabilities, Zscaler Cloud Firewall, Fortinet, Sophos

Advantages: blocks malicious web traffic, protects multiple cloud deployments, improved network visibility, enhanced reliability, easier maintenance

Disadvantages: resistance to acceptance, network latency issues

Desktop-as-a-Service DaaS

Virtual desktops and apps to subscribe, Azure Windows Virtual Desktop, Citrix Managed Desktop

Advantages: global accessability, simplified management, reduced downtime, low cost, high flexibility, high scalability

Disadvantages: security issues, network connectivity issues, high licensing costs

Mobile Backend-as-a-Service MBaaS

Integrate front-end applications with backend infrastructure through API and software development kit SDK, Google Firebase, Apple Cloudkit

Advantages: improved development efficiency, high flexible, scalable, pay-as-you-go

Disadvantages: security issues, high initial costs

Machines-as-a-Service MaaS

aka Equipment-as-a-Service EaaS, allows manufacturers to sell or lease machines to clients and receive a percentage of profits generated by those machines

Advantages: low investment costs, improved adaptability, reliable and cost-effective income source, improved product quality and quantity

Disadvantages: maintenance and repairs are expensive, machines replace human workers, resulting in unemployment

Separation of responsibilities in Cloud

The separation of responsibilities of subscribers and service providers is essential. Separation of duties prevents conflict of interest, illegal acts, fraud, abuse and error and helps in identifying security control failures, including information theft, security breaches and evasion of security controls.

There are mainly three types of cloud services: IaaS, PaaS and SaaS. It is essential to know the limitations of each cloud service delivery model.

Cloud Deployment Models

The five standard cloud deployment models:

Public Cloud

Makes services such as applications, servers and data storage available to the public over the internet; the provider is liable for the creation and maintenance of the public cloud and its IT resources, Google App Engine, Amazon EC2, Windows Azure etc.

Private Cloud

aka Internal cloud, corporate cloud; cloud infrastructure operated by a single organization and with a corporate firewall, VMware vRealize Suite, SAP Cloud Platform etc.

Community Cloud

Multi-tenant infrastructure shared among organizations from a specific community with common computing concerns, Cisco Cloud Solutions, Salesforce Health Control etc.

Hybrid Cloud

Cloud environment comprised of two of more clouds (public, private or community), Microsoft Azure, Zymr, Logicalis; example: an organization performs it's critical activities on the private cloud and non- critical activities on the public cloud

Multi Cloud

Dynamic heterogeneous environment that combines workloads across multiple cloud vendors that are managed via one proprietary interface

Distributed Cloud

Centralized cloud environment comprised of geographically distributed public or private clouds controlled on a single control plane for providing services to the end users located on or off site

Poly Cloud

Several types of cloud services, which can be supplied to different other clouds

Fog Computing

The massive growth in IoT devices worldwide has resulted in the production of an enormous amound of data by these devices. To meet the growing demand for analyzing and processing these data, the implementation of fog computing, along with cloud computing, is an ideal solution. Fog computing is a distributed and independent digital environment in which the applications and data storage are positioned between data sources (devices generating data) and the cloud service. Fog computing is an extended version of cloud computing that comprises multiple edge nodes that are directly connected to physical devices to enable service access to the end users. Generally, fog refers to the idea of creating an individual layer in distributed network infrastructure that has close inter-connections with IoT and cloud computing. It acts as an intermediary between hardware and remote servers, and is also called intelligent gateway.

      Low Latency
      High business agility
      No glitches with bandwidth
      No connection loss
      Elevated security
      Low operation costs
      High power efficiency

Edge computing

Conventional cloud computing has some issues related to data security, low performance and increased data storage which lead to high operational costs. These issues can be solved by replacing conventional cloud computing with edge computing. Edge computing is a subset of fog computing and its approach to data processing is similar to that of fog computing. In fog computing, the intelligent gateway performs processing at the LAN, whereas in edge computing, the edge gateway intelligence is performed in devices such as programmable automation controllers. Edge computing is a distributed, decentralized computing model in which computation and data processing are performed close to edge devices.

Container Technology

Emerging container-based virtualization service. A container is a package of an application/software including all its dependencies, such as library and configuration files, binaries and other resources. All these resource files are delivered as a unit to solve compatibility issues when applications are moved between cloud environments. These containers are provided to subscribers in the form of a CaaS. Popular container service include Amazon AWS EC2, Google Kubernetes Engine GKE, Docker etc.

      Portability and consistency
      Security
      High efficiency and cost effectiveness
      Scalability
      Robustness

To resolve issues of virtualization (slower performance, portability issues etc.) industries are adopting a containerization technology that provides application resources in the form of lightweight containers that run on a single operating system and makes the software/application run anywhere with scalable resources. Containers are placed on top of a physical server and host operating system and share the system kernel binaries and libraries, reducing the need for reproducing the OS. Virtual Machines are heavyweight, Containers are lightweight and portable. Virtual Machines are hardware based virtualizations. Containers are OS based virtualization.

Docker

Docker is an open-source technology used for developing, packaging and running applications. All docker dependencies are in the form of containers to ensure that applications work in a seamless environment. Docker provides a PaaS through OS - level virtualization and delivers containerized software packages. This technology isolates applications from the underlying infrastructure for faster software delivery. Benefits of Docker is that when an application is packaged along with its dependencies in a Docker container, it can run in any environment.

The Docker engine is a client/server application installed on a host that allows to develop, deploy and run applications using:

      Server (daemon process dockerd)
      Rest API
      Client CLI
