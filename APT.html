APT

Advanced Persistent Threat (APT) is a major security concern for any organization. Recently in the news it was revealed that a member of an “APT” group that utilises the “Conti” ransomware became disgruntled at the state of their relationship with the group and leaked a large majority of the groups “Tools, Techniques and Procedures” documents. Conti was first discovered in 2020 and is used primarily by the Russian crime group aliased as “Wizard Spider” (we will assume the leak came from a former member of this group). In addition to the encryption of files seen from historical families of ransomware, Conti also copies files to the attacker’s server before encryption, allowing the group to threaten data leaks if the ransom isn’t paid.

What is it

An APT is defined as a type of network attack, whereby the attacker gain unauthorized access to a target network and remains there undetected for a long time. Advanced: techniques to exploit underlying vulnerabilities, persistent: external command & conquer (C&C) system continuously extracting data and monitoring the victim's network, threat: human involvement in coordination.

APT attacks are highly sophisticated attacks whereby an attacker uses well-crafted malicious code along with a combination of multiple zero-day exploits to gain access to the target network. Attacks involve well planned and coordinated techniques whereby attackers erase evidence of their malicious activities after their objectives have been fulfilled.

Victims

APT attacks are usually performed on organizations related to financial, healthcare, defense, aerospace etc. Main goal is to obtain sensitive information, like:

      classified documents
      user credentials
      employee's or customer's personal information
      network information
      transaction information
      credit card information
      organization's business strategy informaton
      control system access information

Characteristics

      Objectives: repeatedly obtain sensitive information for illegal earnings / spying for political or strategic goals
      Timeliness: time taken by attacker from assessing the target system for vulnerabilities to exploiting them to gain and maintain access
      Resources: amount of knowledge, tools and techniques required to perform an attack. APT attacks are more sophisticated attacks performed by highly skilled cybercriminals with considerable resources
      Risk tolerance: level up to which the attack remains undetected in the target network
      Skills and methods: various social engineering techniques to gather information, techniques to prevent detection by security mechanisms and techniques to maintain access for a long time
      Actions: technical actions to maintain presence in the victim's network for a long time and extract as much data as possible
      Attack originating points: numerous attempts made to gain entry into the network, the attacker needs to conduct exhaustive research to identify vulnerabilities and gatekeeper functions in the target network
      Numbers involved in the attack: number of hosts involved in the attack, usually performed by a crime group / organization
      Knowledge source: gathering of information through online sources about specific threats which can be further exploited to perform certain attacks
      Multi phased: reconnaissance, access, discovery, capture, data exfiltration
      Tailored to the vulnerabilities: malicious code is designed and written such that it targets the specific vulnerabilities present in the victim's network
      Multiple points of entries: in the initial phase of an APT attack the adversary creates multiple points of entry to maintain access, if one point is detected a different entry point can be used
      Evading signature based detection systems: APT attacks are closely related to zero-day exploits, APT attacks can easily bypass security mechanisms
      Specific Warning Signs: APT attacks are usually impossible to detect, warning signs are: presence of a backdoor Trojan for maintaining access, unusual file transfers, unusual database activities etc.

APT Lifecycle

      Preparation: adversary defines target, performs extensive research, organizes a team, build tools and perform tests
      Initial Intrusion: sending spear-phishing emails, exploiting vulnerabilities on publicly available servers, sometimes social engineering techniques, malicious code or malware is deployed to initiate an outbound connection
      Expansion: the primary objectives of this phase are expanding access to the target network and obtaining credentials. In most cases the objective of an attacker is to access multiple systems using a single compromised system. The attacker wants to obtain administrative login credentials to escalate privileges and gain further access to the systems in the network
      Persistence: maintaining access to the target system, evading endpoint security devices like IDS and firewalls, finding locations for installing malware that are not frequently examined, including routers, servers, firewalls, printers etc.
      Search and exfiltration: gaining access to a resource that can be used for performing further attacks or using that resource for financial gain
      Cleanup: prevent detection and remove evidence of compromise / make the system appear as it was before they gained access
